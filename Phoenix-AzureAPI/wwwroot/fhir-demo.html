SQL to FHIR Mapping
SQL Database Field FHIR Resource Property Transformation
PatientID Patient.id and Patient.identifier Used as both the resource ID and as an identifier with system "http://phoenix-azure.org/fhir/identifier/patient-id"
First, Last Patient.name Mapped to a HumanName structure with given and family components
Gender Patient.gender Mapped to FHIR AdministrativeGender code system (male, female, other, unknown)
BirthDate Patient.birthDate Formatted as YYYY-MM-DD string
PatientAddress, City, State, Zip Patient.address Mapped to FHIR Address structure with line, city, state, postalCode components
Phone Patient.telecom Added as ContactPoint with system="phone"
Email Patient.telecom Added as ContactPoint with system="email"
